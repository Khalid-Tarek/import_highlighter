Import Error Highlighter

Marks error rows in an import CSV based on a Maximo import preview.

Symptom

A Maximo import preview reports BMXAA5598E errors with messages like:

  • CTGCC1046I — The CIRelation has not been saved, because the CI must exist before you can save the CIRelation.
  • BMXAA4129E — The record for Source=[CI], Target=[CI], Relation=[REL] already exists.

Scrolling between the preview text and the CSV to find which rows failed is tedious and error-prone, especially on large imports with hundreds of errors.

Resolution

Use the Import Error Highlighter tool:

  1. Paste the entire import preview text (including error messages) into the tool
  2. Select the CSV file you were trying to import
  3. Click Mark Import Errors

The tool parses every BMXAA5598E line, reads the affected CSV line ranges, and writes each error message into a new Import_Error column on the matching row. A .bak backup of the original CSV is saved automatically.

Download & Build

Source and build scripts: https://github.com/Khalid-Tarek/import_highlighter

Double-click build.bat to create a standalone ImportHighlighter.exe (no Python required on the target machine).

Quick Start

  1. Open the tool (run the .exe or python src/import_highlighter.py)
  2. Paste the full import preview text into the large text box
  3. Click Browse for CSV... and pick your import CSV
  4. Click Mark Import Errors
  5. Open the updated CSV — each row with an error has the message in the Import_Error column

Example

Before — CSV has no error column:

CINUM,SOURCECI,RELATIONNUM,TARGETCI
F5VIP-10.19.38.180:443,F5VIP-10.19.38.180:443,RELATION.LOADBALANCES,SERVER-10.19.172.163

After running the tool, the same row includes the error:

CINUM,SOURCECI,RELATIONNUM,TARGETCI,Import_Error
F5VIP-10.19.38.180:443,F5VIP-10.19.38.180:443,RELATION.LOADBALANCES,SERVER-10.19.172.163,"BMXAA4129E - The record for Source=F5VIP-..., Target=SERVER-..., Relation=RELATION.LOADBALANCES already exists."
